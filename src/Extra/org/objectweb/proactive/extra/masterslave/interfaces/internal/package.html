internal Interfaces of the Master/Slave API. The package includes :

  • MasterIntern : an internal version of the Master interface
  • ResultIntern : an oject containing a task and its result
  • Slave : the slave interface
  • SlaveConsumer : from the SlaveManager point of view, the entity that will be notified of new slaves deployment
  • SlaveDeadListener : in relation with the SlaveWatcher(Pinger), the enity that will be notified of slave failure
  • SlaveManager : the interface of the SlaveManager which allows to get slaves from the Slave pool
  • SlaveManagerAdmin : the admin interface of the SlaveManager which allows to increase the slave pool
  • SlaveWatcher : the entity which takes care that slaves are alive (Pinger)
  • TaskIntern : the internal version of the Task
  • TaskProvider : from the slave point of view the entity which provides task and expects results (i.e. the master)
